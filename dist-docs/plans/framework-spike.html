Framework Spike: Exercise Context Card ​

Side-by-side comparison of the Exercise context card in Svelte vs Lit vs current vanilla JS. Covers: card display (summary + health dot), editor modal (select fields, tag fields, note, save/clear), state management, and reactivity.

Current Vanilla JS (~80 lines across two functions) ​

js
// Card rendering (inside renderProfileContextCards loop)
html += `<div class="context-card" onclick="openExerciseEditor()">
  <div class="context-card-header">
    <span class="ctx-health-dot ctx-health-dot-gray" id="ctx-dot-exercise"></span>
    <span class="context-card-label">🏋️ Exercise</span>

</div>`;

// Editor — takes over the shared modal
function openExerciseEditor() {
  const modal = document.getElementById("detail-modal");
  const overlay = document.getElementById("modal-overlay");
  const current = state.importedData.exercise || { frequency: null, types: [], intensity: null, dailyMovement: null, note: '' };
  modal.innerHTML = `<button class="modal-close" onclick="closeModal()">&times;</button>
    <h3>Exercise & Movement</h3>
    <div class="modal-unit">Describe your exercise routine.</div>
    ${renderSelectField('Frequency', 'exercise-freq', EXERCISE_FREQ, current.frequency)}
    ${renderTagsField('Types', 'exercise-types', EXERCISE_TYPES, current.types)}
    ${renderSelectField('Intensity', 'exercise-intensity', EXERCISE_INTENSITY, current.intensity)}
    ${renderSelectField('Daily movement', 'exercise-movement', DAILY_MOVEMENT, current.dailyMovement)}
    ${renderNoteField(current.note)}
    ${contextEditorActions(state.importedData.exercise != null, 'saveExercise', 'clearExercise')}`;
  overlay.classList.add("show");
}

function saveExercise() {
  const frequency = getSelectedOption('exercise-freq');
  const types = getSelectedTags('exercise-types');
  const intensity = getSelectedOption('exercise-intensity');
  const dailyMovement = getSelectedOption('exercise-movement');
  const note = (document.getElementById('ctx-note-input') || {}).value || '';
  if (!frequency && types.length === 0 && !intensity && !dailyMovement && !note.trim()) {
    state.importedData.exercise = null;
  } else {
    state.importedData.exercise = { frequency, types, intensity, dailyMovement, note: note.trim() };
  }
  saveAndRefresh('Exercise saved', 'exercise');
}

Pain points:

  • innerHTML string building (XSS risk, hard to read)
  • Manual DOM queries to read form state (getSelectedOption, getSelectedTags)
  • Shared modal element — openExerciseEditor replaces innerHTML of #detail-modal
  • State read/write through global state.importedData + manual saveImportedData()
  • Health dot update requires separate loadContextHealthDots() call after save
  • window.* exports for onclick handlers

<!-- ExerciseEditor.svelte -->
<script>
  import { importedData } from '$lib/stores/profile'
  import SelectField from '$lib/components/SelectField.svelte'
  import TagsField from '$lib/components/TagsField.svelte'
  import EditorActions from '$lib/components/EditorActions.svelte'
  import { EXERCISE_FREQ, EXERCISE_TYPES, EXERCISE_INTENSITY, DAILY_MOVEMENT } from '$lib/constants'

  export let close

  // Bind directly to reactive state — no manual DOM reads
  let frequency = $importedData.exercise?.frequency ?? null
  let types = $importedData.exercise?.types ?? []
  let intensity = $importedData.exercise?.intensity ?? null
  let dailyMovement = $importedData.exercise?.dailyMovement ?? null
  let note = $importedData.exercise?.note ?? ''

  function save() {
    if (!frequency && !types.length && !intensity && !dailyMovement && !note.trim()) {
      $importedData.exercise = null
    } else {
      $importedData.exercise = { frequency, types, intensity, dailyMovement, note: note.trim() }
    }
    close()
  }

  function clear() {
    $importedData.exercise = null
    close()
  }
</script>

<Modal on:close={close}>
  <h3>Exercise & Movement</h3>
  <p class="modal-unit">Describe your exercise routine.</p>

  <SelectField label="Frequency" options={EXERCISE_FREQ} bind:value={frequency} />
  <TagsField label="Types" options={EXERCISE_TYPES} bind:selected={types} />
  <SelectField label="Intensity" options={EXERCISE_INTENSITY} bind:value={intensity} />
  <SelectField label="Daily movement" options={DAILY_MOVEMENT} bind:value={dailyMovement} />

  <div class="ctx-field-group">
    <label class="ctx-field-label">Notes</label>
    <input type="text" class="ctx-note-input" placeholder="Anything else..." bind:value={note} />

svelte
<!-- SelectField.svelte — reusable pill button selector -->
<script>
  export let label
  export let options
  export let value = null
</script>

<div class="ctx-field-group">
  <label class="ctx-field-label">{label}</label>
  <div class="ctx-btn-group">
    {#each options as option}
      <button
        type="button"
        class="ctx-btn-option"
        class:active={value === option}
        on:click={() => value = value === option ? null : option}
      >{option}</button>
    {/each}
  </div>
</div>

What's better:

  • No innerHTML — template is declarative, auto-escaped
  • bind:value eliminates manual DOM reads — form state is JS variables
  • $importedData store auto-persists (wrapper around localStorage)
  • ContextCard.svelte is reused for all 9 cards — card shell, health dot, AI summary all in one component
  • SelectField/TagsField are reusable across all editors (currently they're functions returning HTML strings — same concept, better ergonomics)
  • Modal is a component, not innerHTML replacement of a shared div
  • No window.* exports, no onclick strings

What's the same:

  • CSS classes unchanged — existing styles.css works as-is
  • Data model unchanged — { frequency, types, intensity, dailyMovement, note }
  • UX unchanged — pills, tags, save/clear/cancel

What's worse:

  • Requires build step (Vite + svelte plugin)
  • Svelte-specific syntax to learn ($: reactive, bind:, #each/#if)
  • Component file proliferation (ContextCard + ExerciseEditor + SelectField + TagsField + EditorActions + Modal = 6 files vs 1)

Lit Version (Web Components) ​

js
// context-card.js — reusable card shell
import { LitElement, html, css } from 'lit'

class ContextCard extends LitElement {
  static properties = {
    emoji: { type: String },
    label: { type: String },
    tooltip: { type: String },
    summary: { type: String },
    placeholder: { type: String },
    healthDot: { type: String },
    aiSummary: { type: String },
  }

  // Use light DOM so existing styles.css applies
  createRenderRoot() { return this }

  render() {
    const filled = !!this.summary
    return html`
      <div class="context-card" @click=${() => this.dispatchEvent(new Event('edit'))}>
        <div class="context-card-header">
          <span class="ctx-health-dot ctx-health-dot-${this.healthDot || 'gray'}"></span>
          <span class="context-card-label">${this.emoji} ${this.label}</span>
          <span class="context-info-icon">i<span class="context-tooltip">${this.tooltip}</span></span>
          <button class="diagnoses-edit-btn"
            @click=${e => { e.stopPropagation(); this.dispatchEvent(new Event('edit')) }}>
            ${filled ? 'Edit' : '+ Add'}
          </button>
        </div>
        ${filled
          ? html`<div class="context-card-body">${this.summary}</div>`
          : html`<div class="context-card-placeholder">${this.placeholder}</div>`}
        ${this.aiSummary
          ? html`<div class="ctx-ai-summary" .innerHTML=${this.aiSummary}></div>`
          : ''}
      </div>
    `
  }
}
customElements.define('context-card', ContextCard)

// exercise-editor.js
import { LitElement, html } from 'lit'
import { state } from '../state.js'
import { EXERCISE_FREQ, EXERCISE_TYPES, EXERCISE_INTENSITY, DAILY_MOVEMENT } from '../constants.js'
import './select-field.js'
import './tags-field.js'

class ExerciseEditor extends LitElement {
  static properties = {
    frequency: { state: true },
    types: { state: true },
    intensity: { state: true },
    dailyMovement: { state: true },
    note: { state: true },
  }

  createRenderRoot() { return this }

  connectedCallback() {
    super.connectedCallback()
    const current = state.importedData.exercise || {}
    this.frequency = current.frequency ?? null
    this.types = [...(current.types || [])]
    this.intensity = current.intensity ?? null
    this.dailyMovement = current.dailyMovement ?? null
    this.note = current.note ?? ''
  }

  save() {
    if (!this.frequency && !this.types.length && !this.intensity && !this.dailyMovement && !this.note.trim()) {
      state.importedData.exercise = null
    } else {
      state.importedData.exercise = {
        frequency: this.frequency,
        types: this.types,
        intensity: this.intensity,
        dailyMovement: this.dailyMovement,
        note: this.note.trim(),
      }
    }
    this.dispatchEvent(new CustomEvent('save', { detail: { field: 'exercise', msg: 'Exercise saved' } }))
  }

  clear() {
    state.importedData.exercise = null
    this.dispatchEvent(new CustomEvent('save', { detail: { field: 'exercise', msg: 'Exercise cleared' } }))
  }

  render() {
    return html`
      <h3>Exercise & Movement</h3>
      <div class="modal-unit">Describe your exercise routine.</div>

      <select-field label="Frequency" .options=${EXERCISE_FREQ} .value=${this.frequency}
        @change=${e => this.frequency = e.detail}></select-field>
      <tags-field label="Types" .options=${EXERCISE_TYPES} .selected=${this.types}
        @change=${e => this.types = e.detail}></tags-field>
      <select-field label="Intensity" .options=${EXERCISE_INTENSITY} .value=${this.intensity}
        @change=${e => this.intensity = e.detail}></select-field>
      <select-field label="Daily movement" .options=${DAILY_MOVEMENT} .value=${this.dailyMovement}
        @change=${e => this.dailyMovement = e.detail}></select-field>

      <div class="ctx-field-group">
        <label class="ctx-field-label">Notes</label>
        <input type="text" class="ctx-note-input" placeholder="Anything else..."
          .value=${this.note} @input=${e => this.note = e.target.value} />
      </div>

      <div class="ctx-editor-actions">
        <button class="import-btn import-btn-primary" @click=${() => this.save()}>Save</button>
        <button class="import-btn import-btn-secondary" @click=${() => this.dispatchEvent(new Event('cancel'))}>Cancel</button>
        ${state.importedData.exercise != null
          ? html`<button class="import-btn import-btn-secondary" style="color:var(--red);border-color:var(--red);margin-left:auto"
              @click=${() => this.clear()}>Clear</button>`
          : ''}
      </div>
    `
  }
}
customElements.define('exercise-editor', ExerciseEditor)

// select-field.js — reusable pill button selector
import { LitElement, html } from 'lit'

class SelectField extends LitElement {
  static properties = {
    label: { type: String },
    options: { type: Array },
    value: { type: String },
  }

  createRenderRoot() { return this }

  toggle(option) {
    const newValue = this.value === option ? null : option
    this.value = newValue
    this.dispatchEvent(new CustomEvent('change', { detail: newValue }))
  }

  render() {
    return html`
      <div class="ctx-field-group">
        <label class="ctx-field-label">${this.label}</label>
        <div class="ctx-btn-group">
          ${this.options.map(o => html`
            <button type="button"
              class="ctx-btn-option ${this.value === o ? 'active' : ''}"
              @click=${() => this.toggle(o)}>${o}</button>
          `)}
        </div>
      </div>
    `
  }
}
customElements.define('select-field', SelectField)

What's better:

  • No innerHTML string building — html tagged template with auto-escaping
  • Components are native Web Components — work in any framework or none
  • Light DOM (createRenderRoot() { return this }) means existing styles.css applies unchanged
  • No build step required — Lit works as a native ES module (import { LitElement, html } from 'lit')
  • Event-based communication (@change, @save, @cancel) — no window.* coupling
  • Internal state via static properties — auto re-renders on change

What's the same:

  • Still JavaScript — no new syntax to learn beyond html tagged templates
  • Same import pattern as current codebase (import ... from './state.js')
  • CSS classes unchanged
  • Data model unchanged
  • Could vendor Lit into vendor/ (it's 6kb)

What's worse:

  • More verbose than Svelte (explicit dispatchEvent, property declarations)
  • No bind: shorthand — two-way binding is manual (@input=${e => this.note = e.target.value})
  • No meta-framework (no SvelteKit equivalent) — routing, SSR would need separate solutions
  • Web Component gotchas (form participation, SSR hydration) though light DOM avoids most
  • CustomEvent/dispatchEvent pattern is noisier than Svelte's bind/event system

Svelte wins on DX — less boilerplate, bind shortcuts, SvelteKit for routing. Best choice if you commit to a full rewrite.

Lit wins on migration path — no build step, same JS, same import patterns, can coexist with existing vanilla modules. You could migrate one component at a time without touching anything else. The web standard angle means zero lock-in risk.

The question is: do you want a clean break (Svelte) or a gradual migration (Lit)?
