Tauri Monorepo Plan ​

Problem ​

xAI and OpenAI APIs don't support CORS — can't call them directly from the browser. Venice and OpenRouter work as proxies but are intermediaries.

Decision ​

Migrate to a monorepo: web PWA + Tauri desktop app sharing the same codebase. Desktop app uses Rust to bypass CORS, enabling direct xAI/OpenAI/Google AI access.

Phased Approach ​

Phase 1: Tauri Wrapper (2-3 sessions) — do this first ​

  • Wrap existing vanilla JS app in Tauri webview (no UI rewrite)
  • Add Rust commands to proxy API calls (bypasses CORS)
  • Unlock xAI, OpenAI, Google AI as direct providers in desktop app
  • GitHub Actions CI builds for Mac, Windows, Linux
  • Auto-updates via Tauri's built-in updater
  • Web version at getbased.health stays unchanged
  • ~5% ongoing overhead (Rust function per new provider, automated release builds)

Phase 2: Svelte UI Migration (10-15 sessions) — later ​

  • Extract schema, constants, data pipeline, API layer to packages/shared as TypeScript
  • Rewrite UI modules as Svelte components (views.js, chat.js, pdf-import.js, settings.js, context-cards.js, emf.js are the big ones)
  • Each ~500-1500 lines of innerHTML/DOM manipulation → reactive Svelte components

Phase 3: SvelteKit Web (1-2 sessions) — last ​

  • Replace vanilla JS web version with SvelteKit static build using the Svelte components from Phase 2

Monorepo Structure ​

packages/
  shared/     — TS types, schema, constants, data pipeline
  ui/         — Svelte components (charts, modals, cards, chat)
  web/        — SvelteKit static PWA
  desktop/    — Tauri (Rust backend + same Svelte UI)

When to Start ​

  • Phase 1: After current feature wave settles (recommendations shipped, Jonseed feedback addressed, i18n queued). Clean 2-3 session project.
  • Phase 2: When vanilla JS actively blocks a feature. Trigger points:
    • i18n gets painful (string interpolation in innerHTML)
    • Cross-device sync needs proper state management
    • Module count passes ~35 and window.* coupling hurts maintenance
  • Phase 3: Once Svelte components are mature enough to replace vanilla JS web version.

Desktop App Benefits ​

  • Direct xAI, OpenAI, Google AI (no CORS)
  • Native keychain for API keys (not localStorage)
  • Real filesystem backup (no File System Access API permission dance)
  • Background sync potential
  • Smaller attack surface
  • Taskbar/dock presence, auto-updates, no browser chrome

Key Insight ​

Phase 1 is additive — no rewrite risk, minimal overhead. The vanilla JS app is still the prototype; every feature shipped now teaches what the real data flow should be. Rewrite when "I can't do X" not "it would be cleaner."
